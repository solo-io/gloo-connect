    Storage-Based API

    QLoo, like Gloo, features a storage-based API. Inspired by Kubernetes, QLoo's API is accessed by applications and users by reading and writing API objects to a storage layer QLoo is configured (at boot-time) to monitor for changes. Currently supported storage backends are Kubernetes CRDs, Consul Key-Value Pairs, or QLoo's local filesystem.
